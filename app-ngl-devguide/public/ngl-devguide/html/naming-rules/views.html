Views

Une vue est une template HTML qui à pour rôle de mettre en forme les données.

1. Les packages pour les ressources

Comme déjà indiqué dans le chapitre précédent, nous développons des applications web qui se focalise sur la ressource. Pour cela nous créons donc un package au nom de la ressource primaire views.<resourcenames>. Si il y a des ressources de sous-niveaux nous créons des sous-packages à partir de celui de la ressource primaire (views.<resourcenames>.<sub-resourcenames>).

L'objectif ici est de gérer les chambres d'un hotel. La ressource primaire est également hotels. 2. Le nom des vues

Le nom du fichier correspondant à la vue doit facilement permettre de comprendre qu'elle est l'objectif de la vue. Le nom des fichiers doit être en anglais.

Dans un package de ressource donnée, il y a 2 catégories de vue :

  • Celles affichants des informations correspondants à la ressource du package
  • Celles affichants des informations correspondants à une ressource d'un autre package

Il y a déjà quelques noms prédéfinies mais la liste çi-dessous n'est pas exhaustive et il faut donc bien se demander qu'elle est la fonction de la vue que je dois développer pour définir le nom du fichier.

2.1. Nom des vues pour la ressource du package

  • home.scala.html : pour le chargement de la configuration d'AngularJS. Page appelée depuis un menu ou une URL.
  • search.scala.html : pour la recherche (formulaire + résultats) de la ressource indiqué par le nom du package
  • details.scala.html : pour afficher les details de la ressource indiquée par le nom du package
  • list.scala.html : pour afficher une liste de la ressource indiquée par le nom du package

2.1. Nom de vues pour une ressource d'un autre package

La règle est simple. Il faut ajouter le nom de la ressource au nom de la vue qui corresponds à sa fonction

  • search<ResourceNames>.scala.html : pour la recherche (formulaire + résultats) de la ressource indiquée par le nom de la vue
  • details<ResourceNames>.scala.html : pour afficher les details de la ressource indiquée par le nom de la vue
  • list<ResourceNames>.scala.html : pour afficher une liste de de la ressource indiquée par le nom de la vue
Remarque : Il n'y a qu'une seule vue home.scala.html par package de ressource.
