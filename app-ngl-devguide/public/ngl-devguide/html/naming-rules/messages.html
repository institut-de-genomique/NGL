Messages

Les messages correspondent au texte de l'application. Une application web est prévu pour être multi-langue. Pour cela on utilise le mécanisme d'internationalisation fournit par le framework Play. Ce mécanisme consiste simplement à faire un fichier messages par langue (messages.fr, messages.en-US, etc.) et d'avoir un fichier par défaut messages.

Le fichier messages peut vite devenir illisible, il faut donc une légère nomenclature pour s'y retrouver. Le fichier messages se compose de clé = valeur. La nomenclature s'applique à la clé. Toutes les clés doivent être en minuscule et en anglais.

Il y a deux cas de figure, soit on développe une fonctionnalité d'une ressource primaire, soit on développe un composant complexe (datatable). Dans les 2 cas, une clé est composée de 3 parties et chaque partie est séparée par un "." :

  1. on indique sur quelle ressource primaire ou sur quel composant.
  2. on indique l'élement HTML (header, title, table, select, etc.) ou la nature (msg, error, etc.).
  3. on précise le champs ou l'information.
Remarque : Il est conseillé de regrouper les messages d'une même ressource ou d'un même composant ensemble.
Remarque : La dernière partie de la clé peut être composé de plusieurs mots séparés par des ".". Ceci permet d'avoir une granularité plus importante. Cela peut être utile si il y a des ressources secondaires.
Attention : Au sein d'une ressource on peut utiliser les clés d'une autre ressource mais si elle change cela impactera également toutes les pages.
Il est conseillé de ne pas surcharger les clés des composants communs interne, il est préférable de demander à faire évoluer le composant pour qu'il soit plus paramétrable ou faire l'évolution vous même.
