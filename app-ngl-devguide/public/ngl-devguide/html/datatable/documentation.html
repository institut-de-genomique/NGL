Présentation: Voir demo

Datatable est un module angularJS qui permet de gérer plus facilement le composant dadatable de HTML. Il ajoute beaucoup de fonctionnalités comme la gestion de l'édition, la sauvegarde, la recupération de données. Pour faciliter son utilisation, il existe une directive. Il suffit de passer à cette directive la configuration souhaitée en JSON, et le datatable se génére automatiquement.

Exemple de datatable simple:

  • - En html:

    	<div datatable="datatable.getConfig()">
	</div>

  • - En javacript:

    • - Initialisation:

      Il faut au préalable injecter le service Datatable dans le controller AngularJS. Ensuite on initialise le datatable:

      $scope.datatable = datatable($scope, $scope.datatableConfig);
    • - Configuration:
      
	$scope.datatableConfig = {
			"columns" : [{	"header":"Code Container", //the title
							"property":"code", //the property to bind
							"id":'test', //the column id
							"edit":false, //can be edited or not
							"hide":true, //can be hidden or not
							"order":true, //can be ordered or not
							"type":"String", //the column type
							"choiceInList":false, //when the column is in edit mode, the edition is a list of choices or not
						},
						{	"header":"test 1", //the title
							"property":"test", //the property to bind
							"id":'test2', //the column id
							"edit":false, //can be edited or not
							"hide":true, //can be hidden or not
							"order":true, //can be ordered or not
							"type":"String", //the column type
							"choiceInList":false, //when the column is in edit mode, the edition is a list of choices or not
						}],
			"compact":false,
			"pagination":{
				active:true
			},		
			"search":{
				url:""
				active:false
			},
			"order":{
				mode:'local', //or 
				active:true,
				by:'ContainerInputCode'
			},
			"remove":{
				active:false,
			},
			"save":{
				active:true,
				mode:'local',
			},
			"hide":{
				active:true
			},
			"edit":{
				active:true
			},
			"messages":{
				active:true
			},
			"extraHeaders":{
				number:0,
				dynamic:true,
			},
			"name":"datatable"
	};

Resultat:

Configuration:

  • - Les colonnes (columns/columnsUrl):

    L'attribut columns dans la configuration représente la liste des colonnes dans le datatable. C'est un tableau.

    • Configuration manuelle :
      [{"header":"Code Container", 
					"property":"code",
					"id":'test',
					"edit":false, 
					"hide":true, 
					"order":true, 
					"type":"String", 
					"choiceInList":false,
					}]
      • header: Le libélé de la colonne
      • property: Le nom de l'objet sur lequel on veut attacher la valeur de la ligne
      • id: L'identifiant (unique) de la colonne
      • edit: Peut on editer cette colonne ? (true/false)
      • hide: Peut on cacher cette colonne ? (true/false)
      • order: Peut on ordonner cette colonne ? (true/false)
      • type: Type de la colonne ("String"/"Number"/"Month"/"Week"/"Time"/"DateTime"/"Range"/"Color"/"Mail"/"Tel"/"Url"/"Date")
      • choiceInList: Dans l'édition, est-ce une liste de choix ou un champ libre ? (true liste de choix/false champ libre)
      • listStyle: Si choiceInList est à true, quel style de liste ? (select ( liste déroulante)/radio (liste radio/multiselect (liste choix multiple)
      • possibleValues: Si choiceInList est à true, c'est la liste des choix possibles pour l'utilisateur sous la forme: "possibleValues":[{"code":"test","name":"test"},{"code":"test2","name":"test2"}]
      • extraHeaders: La liste des extra header, les headers au dessus du header principal auquel la colonne appartient
    • Configuration automatique :

      L'attribut columnsUrl dans la configuration représente la liste des colonnes dans le datatable chargé depuis une url.

      • columnsUrl : URL

  • - Le style des buttons (compact):

    L'attribut compact dans la configuration permet de masquer les libélés des buttons si il est à true

  • - La pagination (pagination):

    L'attribut pagination dans la configuration permet de faire apparaitre les fonctions de pagination du datatable

    • active: Active ou pas la fonctionnalité (true/false)
    • mode: Defini si le datatable doit faire un appel serveur lors du changement de page(remote) ou si il utilise les données déja chargées(local) ("remote"/"local")
    • pageNumber:
    • numberPageListMax: le nombre de page que l'on peut séléctionner dans la barre
    • pageList:
    • numberRecordsPerPage: Le nombre d'élément affiché par page
    • numberRecordsPerPageList: la liste des paginations possible. exemple: [{number:10, clazz:'active'},{number:25, clazz:''}]

  • - La recherche (search):

    L'attribut search dans la configuration permet de faire des recherches sur le serveur et donc de charger des données

    • active: Active ou pas la fonctionnalité (true/false)
    • mode: Defini si le datatable doit faire un appel serveur lors de la recherche(remote) ou si il recherche dans les données déja chargées(local) ("remote"/"local")
    • url: L'url à appeler pour charger les données

  • - Le tri (order):

    L'attribut order dans la configuration permet de definir les options de tri

    • active: Active ou pas la fonctionnalité (true/false)
    • mode: Defini si le datatable doit faire un appel serveur lors du tri(remote) ou si il tri les données déja chargées(local) ("remote"/"local")

  • - Nombre d'éléments (showTotalNumberRecords):

    L'attribut showTotalNumberRecords dans la configuration permet d'afficher le nombre total d'éléments (true/false)

  • - La fonction suppression (remove):

    L'attribut remove dans la configuration permet de géré la fonction de suppression dans le tableau

    • active: Active ou pas la fonctionnalité (true/false)
    • withEdit: Oblige l'utilisateur à être en mode édition sur la ligne avant de pouvoir la supprimer(false) (true/false)
    • showButton: Affiche ou pas le button (true/false)
    • mode: Defini si le datatable doit faire un appel serveur lors du tri(remote) ou si il tri les données déja chargées(local) ("remote"/"local")
    • url: l'url à appelé pour effectuer une supression en mode remote
    • callback: Cette fonction est appelé une fois la suppression effectuée

  • - La fonction édition (edit):

    L'attribut edit dans la configuration permet de géré la fonction d'édition dans le tableau

    • active: Active ou pas la fonctionnalité (true/false)
    • withoutSelect: Oblige l'utilisateur à selectionner les lignes avant de pouvoir editer(false) (true/false)
    • showButton: Affiche ou pas le button (true/false)

  • - La fonction sauvegarde (save):

    L'attribut edit dans la configuration permet de géré la fonction d'édition dans le tableau

    • active: Active ou pas la fonctionnalité (true/false)
    • withoutEdit: Oblige l'utilisateur à selectionner les lignes avant de pouvoir editer(false) (true/false)
    • showButton: Affiche ou pas le button (true/false)
    • mode: Defini si le datatable doit faire un appel serveur lors de la sauvegarde(remote) ou si il sauvegardera en local(local) ("remote"/"local")
    • url: l'url à appelé pour effectuer une sauvegarde en mode remote
    • callback: Cette fonction est appelé une fois la sauvegarde effectuée

  • - La fonction apparition/disparition des colonnes (hide):

    L'attribut edit dans la configuration permet de géré la fonction d'apparition/disparition de colonne

    • active: Active ou pas la fonctionnalité (true/false)

  • - La fonction selection (selection):

    L'attribut cancel dans la configuration permet de géré la fonction de selection

    • active: Active ou pas la fonctionnalité (true/false)
    • showButton: Affiche ou pas le button (true/false)

  • - La fonction annuler (cancel):

    L'attribut cancel dans la configuration permet de géré la fonction d'annulation

    • active: Active ou pas la fonctionnalité (true/false)
    • showButton: Affiche ou pas le button (true/false)

  • - La fonction messages (message):

    L'attribut messages dans la configuration permet de géré la fonction d'internationalisation

    • active: Active ou pas la fonctionnalité (true/false)

  • - La fonction detail (show):

    L'attribut show dans la configuration permet de pouvoir implementer une fonction detail avec les données du tableau.

    • active: Active ou pas la fonctionnalité (true/false)
    • showButton: Affiche ou pas le button (true/false)
    • add(la ligne): La fonction à implementer

  • - La fonction ajout de header (extraHeaders):

    L'attribut extraHeaders dans la configuration permet de pouvoir ajouter autant de header que vous souhaité au dessus du header de base avec le libellé des colonnes

    • number: Le nombre de header en plus que vous souhaitez
    • list: la liste des headers exemple: {0:[{"label":"test","colspan":"1"},{"label":"a","colspan":"1"}],1:[{"label":"test2","colspan":"5"}]
    • dynamic: Si dynamic est a false il va utilisé la "list" sinon il va se générer automatique avec la déclaration des colonnes
